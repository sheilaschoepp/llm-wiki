# audit — Verification Spec (Step 5)

The detailed fact-check spec for `audit` Step 5: the coverage gate, the per-page-type callout checklist, the claim-by-claim axes, and the yes/no audit-assumption framing. Run it for each page in scope (every non-verified page, plus the `*[unverified]*` claims on otherwise-`verified` pages). The mode-scope logic and the unstamped-`verified` guard stay in SKILL.md Step 5.

**Coverage gate.** A `verified` stamp requires confirmed full-text coverage of the raw source, never the wiki source page as a proxy. Confirm coverage with a positive probe, not merely the absence of a symptom: the raw's final page or section is present and intelligible, and body text (not only figure captions) is extractable. If the raw cannot be fully read — file missing or moved, image-only scan, OCR-garbled, truncated extraction, encrypted or password-protected, or paywalled to an abstract — the page must not be promoted to `verified`. Set it `needs-update` with a `needs_update_reason:` naming the coverage failure (e.g. "raw is image-only scan; cannot fact-check"), and surface it as a Critical finding. This mirrors ingest's full-text-coverage gate; audit needs it more, because audit is what writes the `verified` stamp. Before stamping `verified`, also spot-check the `#page=N` deep-links by opening the raw and confirming both that the cited content is there and that N resolves to the printed page the locator claims: read the printed number on the raw's first physical page to derive the offset (`first_printed − 1`), then confirm a cited link's physical page N actually carries printed page M — checking content presence alone passes a self-consistent wrong offset. The single-offset formula assumes one arabic run from the first physical page; on a source with roman-numeral front matter, unnumbered dividers, or several pagination runs (common in books and chapter-split sources audit covers), it breaks — there, locate the cited page's actual physical page directly rather than computing from one offset. For a registered raw, read what each page prints straight off the pagination map (`.claude/skills/lint/pagination-map.md`) — the per-page record `lint`'s `locator_page_mismatch` checks every `p. M` against — rather than deriving `p. M` from a single offset a restarting appendix would defeat. Spot-check every `#page=N` whose offset was computed or changed this run, plus at least one already-present link per page, plus **every locator on a cross-source bullet — one naming two or more sources (`Contradictions`, `Tensions`, `Not This`, `Disconfirming Evidence`): open all of those at the cited page, no sampling.** As a deterministic pre-pass, run `python3 .claude/skills/lint/scripts/cited_figure_check.py "1-wiki"` (with the `llm-wiki` conda env active for PyMuPDF; exit code 3 means PyMuPDF is absent — fall back to the manual sweep and note it) to seed and prioritize the worklist of numeric cross-source citations whose figure sits on none of the bullet's cited pages — lint's opt-in `cited_figure_off_page` backstop, which lint hands to audit to settle which half is wrong. Audit owns running it, as it owns `body_hash.py`; it augments, never replaces, the manual no-sampling sweep — it errs toward "present" and covers only decimals and percentages, so a clean script result never excuses a manual open of a cross-source locator, it only adds suspects. A single correct sample does not certify the rest — the abstract→sec.1 mis-offset class and the cross-source mis-location class (a true claim from source A cited to a locator in source B) are both per-locator, and a mis-located citation hides behind a good sibling when only one link per raw is opened. A page whose cross-source bullets were not locator-verified against their cited pages this run is not stamped on their strength: mark those bullets `*[unverified]*` (the hash excludes them) so the page can still be promoted on its verified content without certifying the unchecked cross-source citations, and the next partial audit re-checks and clears them. A guessed or mis-computed offset (e.g. assuming zero on a source not paginated from 1) is a silent-wrong link, not a verification pass (the same spot-check ingest's verification packet requires). Figure or table *presence* is not evidence *sufficiency*: a claim that draws its value from an image-only or otherwise unreadable figure cannot be certified from the extractable caption alone, even when the page's body text is fully readable and the figure is present at its `#page=N` — treat that claim's evidence region as unreadable (mark it `*[tentative]*` and set the page `needs-update`), independent of whether a bullet drop was contemplated; do not stamp the claim `verified` off the caption. When audit clears an `*[unverified]*` marker, the raw locator that settled it is that page's spot-check; record the confirmed `#page=N` in the report, mirroring ingest's proof-of-read.

**Certify each claim through the tiered independent-refuter gate — this is what earns `verified`.** Audit's fact-check is not a same-agent read: each non-obvious claim is certified by independent refuter subagents at the tier its impact sets, per the shared spec (`.claude/skills/multi-skill/references/verification.md` → Tiered Independent-Refuter Verification; `CLAUDE.md` → Page Status). The broad default is Tier 3 — three refuters, each spawned read-only (Read/Glob/Grep, no Edit/Write) from this top-level audit agent and given the claim and the raw, not the page's framing, each prompted to refute; a summary or generalization claim has each refuter recompute the whole set cold (the summary-claim discipline above), not the cited cell. The claim earns `verified` only on unanimous hold; any refutation, or a "cannot confirm" from an unreadable raw region, is a split — mark the claim `*[unverified]*` / `*[tentative]*` and set the page `needs-update` (the distortion-disposition path), never certify on a split. Obvious / definitional bullets and the author's own marked judgement are Tier 0 (no refuter). These refuters are audit's fan-out: the central post-fan-out pass (SKILL.md Step 7 — locator-format normalization and all shared-file writes applied from the top-level agent, the data-list quorum spawned centrally, never nested) applies unchanged.

Check every callout section against the raw — each faithful to the raw, metadata matching, no load-bearing part dropped or distorted:
- Source pages: TL;DR, Contribution, Key Claims, Evidence, Method, Assumptions, Limitations, Appraisal, Concepts and Entities, Contradictions, Open Questions, Connections.
- Concept/entity pages: Idea, Why It Matters, Not This, Examples, Contradictions, Disconfirming Evidence, Open Questions, Connections, Sources — and the page is one idea, written simply.
- Syntheses: every section, with each part of the answer mapped to the raw sources behind the listed source pages.

**Check claim by claim, not only section by section.** Within each callout, walk every non-obvious claim and verify it on both axes against the raw source: (a) support — it is obvious/definitional, the author's own marked judgement, or inline-cited to a source that genuinely backs it in one of the two canonical citation forms (CLAUDE.md → Source Support And Verification), with the located deep-link's anchor and page checked against the raw, not just present: the structural anchor names the actual section/figure the claim draws from and the physical page N resolves there (compute N from the printed→physical offset, never assume zero on a source not paginated from 1); and (b) faithfulness — its meaning, scope, and strength match what that source says, with no overstatement, narrowing, over-generalization, or misinterpretation (a correct restated number does not make a distorted claim faithful) — and, on a multi-source page especially, no cite-versus-contribute error (a result the source quotes from prior work presented as that source's own contribution) and no cross-source merge (a number, method, or result from a different source imported and attributed to this one), and when the source page carries `frames:` (one or more lenses scoping its summary) faithfulness is judged within the union of those frames — content the page omits because it falls outside every listed frame is in-scope-absent, not a faithfulness gap; and (c) reasoned — a causal, comparative, or evaluative claim states its reason or mechanism, not a bare conclusion (the source's reason, faithful to it, for a source claim; the user's own for own-voice judgement), and a mechanism the source does not give is never fabricated and attributed to it (that is a faithfulness violation, not a clarity nit). Obvious bullets and the author's own marked judgement (`Appraisal`, `Open Questions`, `Assumptions`) need no citation — do not manufacture citation wallpaper; but a sourced `Disconfirming Evidence` or synthesis `Tensions` bullet carries an inline citation like any non-obvious claim, exempt only when it is a purely own observation with no source. The fix in Step 7 is determinate where the fact-check settles it: add the missing inline citation once the raw identifies the supporting source, or correct wording that distorts the source; a non-obvious claim no held source supports is dropped or marked `*[tentative]*` with `needs-update`.

**Verify a summary or generalization claim by recomputing its full basis — never by confirming the cell it cites.** A claim that summarizes, aggregates, or generalizes over a set of values — "X fares worse", "most/all tasks improve", "the Y half degrades", "drops across the board", a mean or median, "on average", "consistently", "in every case" — is not verified by opening the one locator it cites and confirming that cell. The cited cell is the evidence the claim's author selected because it fits, so confirming it is the circular reading that certifies the distortion — this is exactly how the class survives a full audit: the check confirms the cells the bullet cites instead of recomputing the set they generalize over. Reconstruct the whole set the claim ranges over from the raw — every row, column, task, or condition it summarizes — compute the aggregate yourself, and test the claim against that. A claim true of the cited cell but false of the recomputed aggregate — three of six tasks improve while the summary says the half "fares worse"; the qualitative column degrades ~10× more than the numeric one the claim blames — is a faithfulness distortion however well-formed and correctly-numbered the cited cell is. Carry the recomputed aggregate into the finding's yes/no audit assumption and into the Step 7 disposition. This defect concentrates in the cross-source and comparison callouts (`Contradictions`, `Tensions`, `Not This`, `Disconfirming Evidence`), where a one-number generalization about which approach or column fares worse is the characteristic failure.

**Verify a citation at the page it cites, not the document — and not by whether the claim is true.** For a cited claim the question is not "is this claim true?" but "is this claim on the page it cites?" A true claim with a wrong locator is a defect, and a more dangerous one than a false claim: its truth is what makes every reviewer nod it through. So open the raw at the cited physical page N and confirm the claim's content sits *there*. Scope the check to that page; never satisfy it with a document-wide search. A distinctive figure or name often recurs elsewhere in the same document — a different table, an unrelated metric, the bibliography — so grepping the whole raw for the literal returns a hit and "confirms" a bad citation; that false-confirmation path is the exact way a careful-looking verification certifies the error. Presence somewhere in the document is not presence at the cited page. On a cross-source bullet, verify each clause against the source it is cited to — the claim's origin, not the bullet's frame (CLAUDE.md → Source Support And Verification, the multi-source-bullet rule).

**Frame every Critical finding as a yes/no audit assumption** (design decision D 7.13 in `a-archive/reference/llm-wiki-design-decisions.md`). Format: `I assumed {X} because I saw {Y}. Correct?` followed by the proposed fix. This forces the audit pass to expose its load-bearing inference rather than rationalize drift. Example: instead of "page X overstates the gain", write "I assumed page X overstates the gain because the source's `Evidence` says 12-18% but the page's `Idea` bullet says 'large gain'. Correct?" then propose the fix. Apply the same yes/no framing to any Warning finding that will trigger an autonomous structural change in Step 7 (a split, a merge, a source-shaped rewrite, a dropped bullet) — those are exactly the load-bearing inferences most prone to rationalized drift, since audit both diagnoses and acts on them. Info findings, and Warnings that only flag without acting, stay in their plainer narrative form. The assumption framing keeps the load-bearing inference auditable in the report even though audit applies the fix autonomously (Step 7).
